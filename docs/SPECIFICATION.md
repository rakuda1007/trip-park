## 1. 概要・目的

**Trip Park** は、グループで旅行やキャンプを計画・実行するときに、メンバー間で日程調整・目的地共有・旅程管理・立て替え精算を一元的に行うための Web アプリ。

### 解決したい課題

| 課題 | 本アプリでの対応 |
|---|---|
| 日程のすり合わせがチャットで流れる | 日程調整機能で候補日と回答を集約 |
| 重要な連絡が埋もれる | 掲示板でピン留め・カテゴリ分け |
| 目的地・経路が分散する | 目的地・経由地・ルートをグループ内で共有 |
| 立て替えの清算が複雑 | 支出登録と精算計算（誰が誰にいくら） |
| 旅行のたびにメンバー情報を再入力 | 世帯マスタで一度登録すれば使い回せる |
| 招待リンクを踏んだ未登録者が参加できない | 招待ランディングページで誘導 |

---

## 2. 想定ユーザー

- **主催者（オーナー）**: 旅行企画、日程・目的地の確定、グループ管理
- **参加者（メンバー）**: 日程回答、情報確認、支出登録
- **複数世帯の参加**: 4人家族が3人だけ参加するケースにも対応
- **毎回異なるメンバー構成**: 「三田小OB旅行」と「宿川原小OB旅行」は別々に管理

## 3. 概念階層（全体構造）

```
ユーザー (User)
  ├─ 世帯マスタ (Household Master)
  │    └─ 奥田家: 大人2・子供2・子供負担比率0.5
  │    └─ （複数登録可能）
  │
  └─ サークル (Circle)  ← 招待相手の名簿
       └─ 三田小学校の集まり: A,B,C,D,E
       └─ 宿川原小学校の集まり: X,Y,Z

旅行 (Trip)  ← 現在の「グループ」を改称
  ├─ フェーズ: 計画中 / 確定 / 終了
  ├─ 参加世帯（世帯内参加者）
  │    └─ 奥田家: 大人2・子供1（今回3人参加）
  │    └─ 鈴木さん: 大人1（個人参加）
  └─ 機能: 日程調整 / 目的地投票 / 旅程 / 支出・精算 / 掲示板
```

### サークルと旅行の関係

- サークル = 「いつも一緒に旅行する仲間」の名簿（旅行とは独立）
- 旅行作成時にサークルを選ぶと、そのメンバーへ一括で招待を出せる
- 実際に参加したメンバーだけが旅行のメンバーになる

```
三田小の集まり: A,B,C,D,E  ── 案内を出す ──▶  旅行「箱根合宿」
                                                  参加: A,B,C（D,Eは不参加）

宿川原の集まり: X,Y,Z  ── 案内を出す ──▶  旅行「富士山登山」
                                              参加: X,Z（Yは不参加）
```

## 4. 用語定義

| 用語（表示名） | データ上の名称 | 意味 |
|---|---|---|
| **旅行** | Trip / Group | 1回の旅行イベント。1レコード＝1旅行 |
| **旅行名** | name | ユーザーが付ける名前（例: 箱根合宿） |
| **目的地** | destination | 確定した目的地名 |
| **旅行日程** | tripStartDate / tripEndDate | 旅行の期間 |
| **サークル** | Circle | 招待相手の名簿。オーナーのみ閲覧 |
| **世帯マスタ** | Household | ユーザーレベルで登録する家族構成の原本 |
| **参加世帯** | Family（旅行サブコレクション） | この旅行に参加する世帯と人数の記録 |
| **世帯内参加者** | — | 世帯のうち今回の旅行に参加する人数 |
| **同行グループ名** | Family.name | 精算の単位となる名前（例: 奥田家） |

## 5. 招待・参加フロー

### 現状の問題

未ユーザーが招待リンク `https://app.com/join?code=ABCD1234` を踏むと、`AuthGuard` が `/login` にリダイレクトする際に招待コードが消える。ログイン後は `/dashboard` に飛ぶため参加できない。

### 改善後のフロー

```
招待リンク: https://app.com/welcome?code=ABCD1234

① 未ユーザーがタップ
  /welcome?code=... （ログイン不要で表示）
  「〇〇さんから「箱根合宿」への招待が届いています」
    [ログインして参加] → /login?returnTo=/join?code=ABCD1234
    [新規登録して参加] → /signup?returnTo=/join?code=ABCD1234
  ↓
  認証後、returnTo を読んで /join?code=... に戻る
  ↓
  自動的に旅行に参加 → 旅行ホームへ

② Trip Parkユーザー（ログイン済み）がタップ
  /join?code=... で即参加（変更なし）
```

### サークルからの招待フロー

```
旅行作成 → 「サークルから招待」を選択
  → 三田小学校の集まりを選択
  → メンバー一覧（B,C,D,E）が表示
  → 全員 or 個別選択で「招待リンクを発行」
  → 各人にリンクをシェア（LINE・メールなど）
  → 参加した人だけが旅行のメンバーになる
```

## 6. ログイン後のUX

### 自動遷移

```
ログイン
  ↓
localStorage から lastTripId を読み出し
  ├─ 旅行あり → 旅行ホーム（/trips/{tripId}）に直接遷移
  └─ 旅行なし → 「旅行を作成 or 招待で参加」画面
```

- `lastTripId` は `localStorage` に `trip-park:lastTripId:{uid}` のキーで保存
- 旅行ホームを開くたびに自動更新

### ヘッダー構成

```
[ Trip Park ]  [ 箱根合宿 ▼ ]  [ プロフィール ]  [ ログアウト ]
                     ↑
               旅行セレクター（クリックでドロップダウン）
               セクション:
                 • 旅行中
                 • 今後の旅行（残り日数順、直近をハイライト）
                 • 日程未定
                 • 過去の旅行（折りたたみ）
```

## 7. 旅行のフェーズ

### フェーズ1: 計画中（Planning）

> 参加者を集め、日程・目的地を決める段階

表示例: `計画中 〜 2026年GW旅行`

使える機能:
- 参加者招待（招待リンク / サークルからの一括招待）
- 日程調整（○△×回答）
- **目的地候補の投票**（新機能）
- 掲示板

非表示（まだ使わない）: 旅程管理・支出・精算

### フェーズ2: 旅行確定（Confirmed）

> 参加者・日程・目的地が確定した段階

表示例:
```
箱根合宿
箱根 ｜ 2026年8月3日（月）〜5日（水）
```

使える機能:
- 旅程管理（目的地・経由地）
- 参加世帯の登録（世帯マスタから選んで人数指定）
- 支出・精算（世帯名単位）
- 掲示板

フェーズ移行: オーナーが「旅行を確定する」ボタンを押す（手動）

### フェーズ3: 旅行終了（Completed）

> 旅行が終わった状態

- 旅行終了日の翌日以降に「過去の旅行」セクションへ移動
- 全機能が読み取り専用
- 精算レポートの最終確認

## 8. 機能詳細

### 8.1 日程調整

- **候補日**: 期間（開始日〜終了日）で登録。1日のみも可
- **回答**: ○ / △ / × の3段階
- **集計表**: メンバー × 候補日のマトリクス表示
- **確定**: オーナー/管理者が候補から1つを選んで確定
- **自動連携**: 日程確定時に `GroupDoc.tripStartDate/tripEndDate` を自動更新

### 8.2 目的地候補の投票（新機能）

計画中フェーズのみ表示。

- 誰でも候補を追加可能
- 全員が各候補に「行きたい / まあいい / 行きたくない」で投票
- 集計バーで支持度を可視化
- オーナー/管理者が「この目的地に決定」→ `destination` フィールドに保存
- 確定後は読み取り専用

### 8.3 旅程管理（目的地・経由地）

- **旅程ブロック**: 見出し・目的地名・住所・メモ・地図リンクで構成
- **経由地**: 旅程ブロックごとに順序付きで登録
- **ルートラベル**: 複数車両・行き帰りを区別するラベル（例: A車・行き）
- **ナビ連携**: 登録済み経由地＋目的地から Google Maps ナビリンクを自動生成
- **地図表示**: `@react-google-maps/api` でインライン表示

### 8.4 掲示板

- **投稿**: タイトル・本文・カテゴリ・重要度
- **カテゴリ**: 全体連絡 / 持ち物 / 当日の連絡 / その他
- **ピン留め**: 管理者が重要な投稿を上部に固定
- **返信**: スレッド形式

### 8.5 世帯マスタ

旅行に関係なく、ユーザーが一度登録すれば使い回せる。

- **登録場所**: プロフィール設定内（`/profile/households`）
- **登録内容**: 世帯名・大人数・子供数・子供負担割合・紐付けるTrip Parkアカウント
- **複数登録可**: 自分の家族・親の家族など複数の世帯を持てる

### 8.6 参加世帯の登録（世帯内参加者）

旅行確定フェーズで使用。

```
操作フロー:
  「参加世帯を追加」ボタン
  → 「世帯マスタから選ぶ」→ 奥田家（大人2・子供2）が自動入力
  → 今回の参加人数を指定: 大人2・子供1（お留守番1人）
  → 保存 → 旅行の参加世帯として登録
```

- **精算単位**: 世帯名（例: 奥田家）単位で精算結果を表示
- **1人参加**: 「個人参加」として登録（世帯マスタ不要）

### 8.7 支出・精算

- **支出登録**: 金額・支払い世帯・日付・カテゴリ・メモ
- **カテゴリ**: 食費 / 交通 / 宿泊 / その他
- **負担の割り方**:
  - 均等割り: 参加世帯の頭数で均等
  - 比率指定: 子供の負担割合を考慮した重み付き
- **精算アルゴリズム**: 貪欲法による送金回数最小化
- **出力**: 「奥田家は田中さんに ¥3,200 支払う」形式の送金指示リスト
- **単位**: 世帯名（同行グループ名）単位で集約

### 8.8 サークル（連絡先グループ）

- **作成**: オーナーが任意のサークルを作成
- **メンバー管理**: 表示名・Trip Parkアカウント（紐付き任意）・メモを登録
- **プライバシー**: サークルはオーナーのみ閲覧可能
- **旅行作成時の連携**: サークルを選択して一括招待リンク発行

## 9. データモデル

```typescript
// ─────────────────────────────────────────────
// 世帯マスタ（ユーザーレベル）
// ─────────────────────────────────────────────
// users/{uid}/households/{householdId}
type HouseholdDoc = {
  name: string;                  // "奥田家"
  defaultAdultCount: number;     // デフォルト大人数
  defaultChildCount: number;     // デフォルト子供数
  defaultChildRatio: number;     // 子供負担比率（大人1=1.0）
  memberUserIds: string[];       // 同じ世帯の他Trip Parkアカウント
  createdAt: Timestamp;
  updatedAt: Timestamp;
};

// ─────────────────────────────────────────────
// サークル（ユーザーレベル）
// ─────────────────────────────────────────────
// circles/{circleId}
type CircleDoc = {
  name: string;                  // "三田小学校の集まり"
  ownerId: string;
  createdAt: Timestamp;
  updatedAt: Timestamp;
};

// circles/{circleId}/members/{memberId}
type CircleMemberDoc = {
  displayName: string;           // "Bさん"
  userId: string | null;         // Trip Parkアカウント（未登録はnull）
  note: string | null;           // メモ（連絡先など）
  addedAt: Timestamp;
};

// ─────────────────────────────────────────────
// 旅行（現在の「グループ」）
// ─────────────────────────────────────────────
// groups/{groupId}
type GroupDoc = {
  name: string;                  // 旅行名
  description: string | null;
  destination: string | null;    // 確定した目的地 ★NEW
  status: "planning" | "confirmed" | "completed"; // フェーズ ★NEW
  circleId: string | null;       // 元になったサークル ★NEW
  ownerId: string;
  inviteCode: string;
  tripStartDate: string | null;  // YYYY-MM-DD
  tripEndDate: string | null;
  createdAt: Timestamp;
  updatedAt: Timestamp;
};

// ─────────────────────────────────────────────
// 参加世帯（旅行サブコレクション、現在の families）
// ─────────────────────────────────────────────
// groups/{groupId}/families/{familyId}
type FamilyDoc = {
  name: string;                  // 世帯名（精算単位の表示名）
  householdMasterId: string | null; // 世帯マスタへの参照 ★NEW
  adultCount: number;            // 今回の旅行での参加大人数
  childCount: number;            // 今回の旅行での参加子供数
  childRatio: number;            // 子供の負担比率
  memberUserIds: string[];       // 紐付けるTrip Parkアカウント
  createdByUserId: string;
  createdAt: Timestamp;
  updatedAt: Timestamp;
};

// ─────────────────────────────────────────────
// 目的地投票（旅行サブコレクション）★NEW
// ─────────────────────────────────────────────
// groups/{groupId}/destinationCandidates/{id}
type DestinationCandidateDoc = {
  name: string;                  // "沖縄"
  description: string | null;   // "費用目安: 1人3万円"
  proposedByUserId: string;
  proposedByDisplayName: string | null;
  createdAt: Timestamp;
};

// groups/{groupId}/destinationVotes/{userId}
type DestinationVoteDoc = {
  candidateId: string;
  answer: "want" | "ok" | "no"; // 行きたい / まあいい / 行きたくない
  updatedAt: Timestamp;
};

// ─────────────────────────────────────────────
// その他（既存・変更なし）
// ─────────────────────────────────────────────
// groups/{groupId}/members/{userId}          → MemberDoc
// groups/{groupId}/scheduleCandidates/{id}   → ScheduleCandidateDoc
// groups/{groupId}/scheduleResponses/{id}    → ScheduleResponseDoc
// groups/{groupId}/config/schedule           → ScheduleConfigDoc
// groups/{groupId}/bulletinPosts/{id}        → BulletinTopicDoc
// groups/{groupId}/bulletinPosts/{id}/replies/{id} → BulletinReplyDoc
// groups/{groupId}/tripRoutes/{id}           → TripRouteDoc
// groups/{groupId}/expenses/{id}             → ExpenseDoc
// inviteCodes/{code}                         → InviteCodeDoc
// users/{uid}                                → UserProfile
// users/{uid}/groups/{groupId}               → UserGroupRefDoc
// users/{uid}/households/{id}                → HouseholdDoc ★NEW
```

## 11. 非機能要件

| 項目 | 内容 |
|---|---|
| 対応デバイス | スマートフォン優先のレスポンシブ Web |
| 認証 | Firebase Authentication（メール/パスワード） |
| データベース | Firebase Firestore |
| プライバシー | 旅行は非公開。招待されたユーザーのみ閲覧可 |

| 地図API | `@react-google-maps/api`（APIキー管理に準拠） |
| セキュリティ | Firestore Security Rules で権限制御 |
